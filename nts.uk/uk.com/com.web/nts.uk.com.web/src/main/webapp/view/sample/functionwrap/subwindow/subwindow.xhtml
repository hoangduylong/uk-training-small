SubWindow

Overview

A SubWindow is a control which open another page with given URL. Basically it's a dialog with <iframe> inside.






Don't style functions-area on your own
- Use id="functions-area" for function area on top
- Use id="functions-area-bottom" for function area on bottom Close button and Title!
- Hide or show close button is decided by Kiban team so don't try to change it. Please ignore design document if there is any different
- Dialog title will be auto display by Kiban team so don't mind about that too

Information

Module: nts.uk.ui.windows.sub
File: /lib/nittsu/ui/ui.ts

Methods

Name

Description
modal(url, option?) Open a modal sub-window, which you can't interact with parent window.
Return: ScreenWindow
Agruments:
url: string Relative URL usually start with "/view"
option: object Please see bellow for Option Object.
modeless(url, option?) Open a modal sub-window, which you can interact with parent window.
Return: ScreenWindow
Agruments:
url: string Relative URL usually start with "/view"
option: object Please see bellow for Option Object.

The Option Object

Property

Type

Default

Description
autoResize Boolean True If set to true, the dialog will be auto resized when the window's size change.
resizable Boolean False If set to true, the dialog will be resizable.
width Number 300 The width of the dialog, in pixels.
height Number Auto The height of the dialog.
There is another way to set width and height of a dialog.
Define a object with name is dialogSize and properties are width and height in global context of dialog.

Note: The way that set width and height to Option Object will be given priority
